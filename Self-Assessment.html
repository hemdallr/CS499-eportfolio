Professional Self-Assessment

Throughout my time in the Computer Science program with SNHU I have been able to develop my professional and technical skills. I have always been interested in technology and business. I knew I wanted to combine the two which is why I received an associate's degree in business administration from another school and am now receiving my bachelor's in computer science with a focus in data analytics. Because of this I wanted to work with data but wasn’t confident enough in myself to learn technical skills. Throughout my time at SNHU I gained a lot of experience in software development, algorithms, and databases. Once I started working with Python, SQL, databases, and analytics, I became more interested in data engineering. As I took my classes and started to learn about machine learning and AI systems, I became more interested in that as well and keeping my options open on trying to pursue a career in AI engineering. I believe that the skills and knowledge I gained throughout my courses and the capstone and ePortfolio show me how much I have developed over the last two years.

When it came to collaborating most of my coursework in the program was completed independently, collaboration and proper communication were always emphasized. There were courses such as Software Development Lifecycle that required communication and collaboration, as it went through the stages it emphasized the importance of communication between the team members to complete a project’s lifecycle. A lot of collaborating happened within the discussion posts between my peers and I where topics and feedback were discussed. I also collaborated with my instructor through their helpful feedback to improve my technical work which in many cases showed me either alternative ways or discuss things that I may have overlooked such as where I can create methods to be able to clean my code and make it reusable and maintainable. This isn't exactly as working in a team environment, it showed me the importance of communication and being adaptable.

Communication is a very important skill to have and develop. Technical skills are great, but it is not enough if you cannot communicate those solutions. I have tried my best to explain my projects and reports as simple as I could even though I know my professors would understand technical terminology, I continued to try to practice simple communication. This is because as I try to get into the field of data analytics, I will have to talk with stakeholders and explain the project and findings through documentation and data visualizations. Someone that inspires how I try to approach communication when it comes to complex technical topics is Neil deGrasse Tyson, because he takes difficult astrophysics topics and simplifies them to the best of his ability. I really learned the importance of communicating to stakeholders in my data analysis class and having to balance the technical side with the non-technical side and present the visualizations, data, design decisions, challenges, and results to stakeholders. The projects in that class strengthened my ability to communicate those technical details and results in a way the stakeholders will understand.

A valuable concept that I learned and developed in this program was data structures and algorithms. Earlier classes such as data structures and algorithms really helped me to understand fundamental structures like linked lists, binary search trees, and hash tables. It also helped me understand the time complexity. In projects, I was able to show my understanding of how choosing the appropriate structure can improve efficiency and performance. I was able to demonstrate this in my capstone project when taking my item tracker project that was written in C++ using a map structure and convert it to Python and use a hash table-based dictionary which improved the average lookup time from O(log n) to O(1). Being able to analyze and know the tradeoffs between algorithms helped to strengthen my problem-solving ability and critical thinking as these were some of the more challenging concepts that I have learned. I also learned that the fastest data structure doesn't always mean the best choice for the project, which taught me about taking into consideration scalability, efficiency, and maintainability.

Throughout the program, my skills with software engineering and databases have strengthened significantly. I was exposed to many different concepts and languages such as C++, Python, Java, and JavaScript. I also strengthened my skills with experiences using object-oriented programming, software design, debugging, and testing. In my first artifact, I took my item tracker project which is in C++ using an object-oriented design and converting it to Python still using an object-oriented design. Beyond my artifacts, my Full Stack class helped me to understand software architecture with the user of Angular, Express, MongoDB, and RESTful APIs. When it comes to databases, this is what really helped influence my decision to pursue data engineering. Through the program I learned a lot about database management through my projects that required me to use SQL, SQLite, MySQL, MongoDB, and Python. This grew my skills in designing databases, data cleaning, querying, and performing proper analysis. I learned how important data quality is and how it can affect businesses and their consumers.

Throughout the program, it also increased my understanding of security through courses such as Software Security, Software Development Lifecycle, and Software Test and Automation QA. Security is not only restricting system access or password protection, but also validating input, managing authentication, and designing and testing secure code. When going through my full stack class, I also learned and was able to implement JSON Web Token authentication and securing API endpoints on the website I developed. This showed me that security must be considered throughout the design and implementation stages rather than after the development cycle.

The artifacts I included in this ePortfolio were to demonstrate my ability, knowledge, and skills throughout my time in the Computer Science program. These artifacts represent my skills in software engineering, algorithms and data structures, and databases. With these artifacts, it allows me to demonstrate my growth and my ability to improve existing solutions, evaluate my previous work, identify improvements, and implement changes.

My first two artifacts are from my CS 210 class using my Item Tracker project which highlights software engineering and design, and algorithms and data structures. I redesigned and improved the project while keeping the original functionality. The algorithm and data structure enhancement highlight my ability to choose an appropriate data structure to improve efficiency and overall performance. The last artifact is databases which I enhance through data processes and visualization. Together, these artifacts demonstrate the skills that I have gained throughout my time in the program and demonstrate the skills I need to pursue a career in data engineering.
